    REST Endpoints¶

    Complete reference for all REST API endpoints in the Spring Security Reference project. Each endpoint demonstrates different authentication and authorization patterns.

    üîì Public Endpoints¶

    GET /api/public/hello¶

    Public endpoint requiring no authentication.

    GET /api/public/hello
Host: localhost:8080

    "Hello, world! (public endpoint - no authentication required)"

    curl http://localhost:8080/api/public/hello

    üéì Learning Points: - No security constraints applied - Accessible without any credentials
    - Used for health checks and public information

    üîê Authentication Endpoints¶

    POST /api/auth/login¶

    Generate JWT token for API authentication.

    POST /api/auth/login
Content-Type: application/x-www-form-urlencoded

username=admin&password=password

    {
  "token": "eyJhbGciOiJIUzUxMiJ9.eyJzdWIiOiJhZG1pbiIsInJvbGUiOiJST0xFX0FETUlOIiwiaWF0IjoxNjk0NTIwMDAwLCJleHAiOjE2OTQ2MDY0MDB9.signature",
  "username": "admin", 
  "role": "ROLE_ADMIN",
  "message": "Login successful - use this JWT token for authenticated requests",
  "usage": "Add header: Authorization: Bearer eyJhbGciOiJIUzUxMiJ9..."
}

    curl -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "username=admin&password=password"

    üéì Learning Points: - Generates JWT tokens with user identity and role claims - No authentication required to obtain token - Token expires after 24 hours (configurable) - Role determination based on username pattern

    üìù Available Test Users: - admin / password ‚Üí ROLE_ADMIN - user / password ‚Üí ROLE_USER - jdbcadmin / password ‚Üí ROLE_ADMIN - ldapadmin / password ‚Üí ROLE_ADMIN

    GET /api/auth/info¶

    Retrieve current authentication information.

    GET /api/auth/info
Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...

    {
  "authenticated": true,
  "username": "admin",
  "authorities": [
    {
      "authority": "ROLE_ADMIN"
    }
  ],
  "authType": "JWT",
  "principalType": "UsernamePasswordAuthenticationToken"
}

    curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  http://localhost:8080/api/auth/info

    üéì Learning Points: - Requires valid authentication - Shows current security context details - Identifies authentication method used - Useful for debugging authentication issues

    üë®‚Äçüíº Admin Endpoints¶

    GET /api/admin/secure¶

    Secure endpoint requiring ROLE_ADMIN authority.

    GET /api/admin/secure
Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...

    {
  "message": "Hello, Admin! (secured endpoint)",
  "user": "admin",
  "authorities": [
    {
      "authority": "ROLE_ADMIN" 
    }
  ],
  "authType": "JWT"
}

    curl -H "Authorization: Bearer YOUR_ADMIN_JWT_TOKEN" \
  http://localhost:8080/api/admin/secure

    üéì Learning Points: - Requires ROLE_ADMIN authority - Returns 403 Forbidden for non-admin users - Supports all authentication methods (JWT, Basic, OAuth2) - Demonstrates role-based access control

    ‚ö†Ô∏è Error Response (403 Forbidden): 

    {
  "error": "Access Denied",
  "message": "You don't have permission to access this resource",
  "timestamp": "2024-01-15T10:30:00Z",
  "path": "/api/admin/secure"
}

    üë§ User Endpoints¶

    GET /api/user/secure¶

    Secure endpoint for users with ROLE_USER or ROLE_ADMIN.

    GET /api/user/secure  
Authorization: Bearer eyJhbGciOiJIUzUxMiJ9...

    {
  "message": "Hello, User! (secured endpoint)",
  "user": "user",
  "authorities": [
    {
      "authority": "ROLE_USER"
    }
  ],
  "authType": "JWT"
}

    curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  http://localhost:8080/api/user/secure

    üéì Learning Points: - Accepts both ROLE_USER and ROLE_ADMIN - Demonstrates role hierarchy (ADMIN > USER) - Multi-role endpoint access pattern - Role-based response customization

    üóÑÔ∏è JDBC Authentication Endpoints¶

    GET /api/jdbc/users¶

    Demonstration endpoint for JDBC database authentication.

    GET /api/jdbc/users
Authorization: Basic amRiY2FkbWluOnBhc3N3b3Jk

    {
  "message": "JDBC Authentication Demo",
  "user": "jdbcadmin",
  "credentials": {
    "jdbcadmin": "password (ROLE_ADMIN)",
    "jdbcuser": "password (ROLE_USER)"
  }
}

    # Using Basic Auth with JDBC credentials
curl -H "Authorization: Basic amRiY2FkbWluOnBhc3N3b3Jk" \
  http://localhost:8080/api/jdbc/users

# Or with username:password
curl -u jdbcadmin:password \
  http://localhost:8080/api/jdbc/users

    üéì Learning Points: - Demonstrates HTTP Basic authentication - Uses database-stored user credentials - Profile-specific endpoint (jdbc profile) - Base64 encoded credentials in Authorization header

    üìù JDBC Test Credentials: - Username: jdbcadmin, Password: password (ROLE_ADMIN) - Username: jdbcuser, Password: password (ROLE_USER)

    üè¢ LDAP Authentication Endpoints¶

    GET /api/ldap/users¶

    Demonstration endpoint for LDAP directory authentication.

    GET /api/ldap/users
Authorization: Basic bGRhcGFkbWluOnBhc3N3b3Jk

    {
  "message": "LDAP Authentication Demo",
  "user": "ldapadmin", 
  "credentials": {
    "ldapadmin": "password (ROLE_ADMIN)",
    "ldapuser": "password (ROLE_USER)"
  }
}

    # Using Basic Auth with LDAP credentials
curl -H "Authorization: Basic bGRhcGFkbWluOnBhc3N3b3Jk" \
  http://localhost:8080/api/ldap/users

# Or with username:password  
curl -u ldapadmin:password \
  http://localhost:8080/api/ldap/users

    üéì Learning Points: - Demonstrates LDAP directory integration - Uses embedded LDAP server for testing - Profile-specific endpoint (ldap profile) - Group-based role mapping

    üìù LDAP Test Credentials: - Username: ldapadmin, Password: password (ROLE_ADMIN) - Username: ldapuser, Password: password (ROLE_USER)

    üåê OAuth2 Authentication Endpoints¶

    GET /api/oauth2/profile¶

    User profile endpoint for OAuth2/OIDC authenticated users.

    GET /api/oauth2/profile
Cookie: JSESSIONID=ABC123...

    {
  "message": "OAuth2 Authentication Demo",
  "user": "john.doe@example.com",
  "email": "john.doe@example.com",
  "provider": "OAuth2",
  "attributes": {
    "sub": "12345",
    "name": "John Doe", 
    "email": "john.doe@example.com",
    "picture": "https://avatar.url"
  }
}

    {
  "message": "OAuth2 Authentication Demo",
  "user": "admin",
  "authorities": [
    {
      "authority": "ROLE_ADMIN"
    }
  ]
}

    # After OAuth2 login, session cookie is used
curl -H "Cookie: JSESSIONID=YOUR_SESSION_ID" \
  http://localhost:8080/api/oauth2/profile

    üéì Learning Points: - Handles both OAuth2 and traditional users - Extracts OAuth2 user attributes (email, name, picture) - Session-based authentication post-OAuth2 flow - Provider-specific attribute handling

    üîß OAuth2 Configuration: - Supports Google, GitHub, Facebook providers - OIDC (OpenID Connect) compatible - Profile-specific endpoint (oauth2 profile)

    üéØ Testing Strategies¶

    Unit Testing Endpoints¶

    @Test
@WithMockUser(roles = "ADMIN")
void adminEndpointWithAdminRole() throws Exception {
    mockMvc.perform(get("/api/admin/secure"))
        .andExpect(status().isOk())
        .andExpect(jsonPath("$.message").value("Hello, Admin! (secured endpoint)"));
}

    Integration Testing¶

    @Test
void jwtAuthenticationFlow() {
    // 1. Login to get JWT token
    String token = getJwtToken("admin", "password");

    // 2. Use token to access secured endpoint
    given()
        .header("Authorization", "Bearer " + token)
        .when()
        .get("/api/admin/secure")
        .then()
        .statusCode(200);
}
